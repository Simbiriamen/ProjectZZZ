#### 2. Новая структура папок (Clean Architecture)
```
src/
├── domain/                     # Доменная область (бизнес-логика)
│   ├── entities/              # Бизнес-объекты (Client, SKU, Recommendation)
│   ├── repositories/          # Интерфейсы репозиториев
│   ├── services/              # Доменные сервисы
│   ├── protocols/             # Протоколы (интерфейсы)
│   └── exceptions/            # Доменные исключения
│
├── application/               # Слой приложения (use cases)
│   ├── commands/              # Команды (CQRS)
│   ├── queries/               # Запросы (CQRS)
│   └── handlers/              # Обработчики команд и запросов
│
├── infrastructure/            # Инфраструктурный слой
│   ├── database/              # Подключение к БД, репозитории
│   ├── persistence/           # ORM модели
│   ├── config/                # Конфигурация и DI контейнер
│   ├── ml/                    # ML модели и предсказания
│   ├── cache/                 # Кэширование
│   └── export/                # Экспорт данных
│
├── interfaces/                # Слой интерфейсов
│   ├── api/                   # REST API endpoints
│   ├── cli/                   # CLI команды
│   └── gui/                   # GUI (если есть)
│
└── shared/                    # Общие утилиты
    ├── di_container.py        # DI контейнер (устаревший, переносится)
    └── ...
```

### Проблемы и решения

#### Проблема 1: Синтаксис pytest.ini
**Симптом:** Ошибка парсинга конфигурационного файла  
**Причина:** Использование формата TOML (`[tool.pytest.ini_options]`) в INI файле  
**Решение:** Переписано в стандартном формате INI (`[pytest]`)

#### Проблема 2: Отсутствие SQLAlchemy
**Симптом:** ImportError при импорте модулей БД  
**Причина:** Не установлена в виртуальном окружении  
**Решение:** Добавлена в зависимости через pip
