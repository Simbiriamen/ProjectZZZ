# ProjectZZZ: Система рекомендаций SKU для менеджеров

**Версия:** 3.1 (Модульная архитектура)  
**Язык:** Python 3.9+  
**Статус:** Готов к разработке/эксплуатации

---

## 📋 Описание

ProjectZZZ — это система машинного обучения для формирования персонализированных предложений из 5 SKU для каждого клиента на плановый визит менеджера. Система использует исторические данные продаж для прогнозирования вероятности покупки и применяет бизнес-правило **2+2+1** для балансировки между новыми товарами, развитием и возвратом клиентов.

### Ключевые возможности

- 🎯 **Персонализация:** Индивидуальный подбор 5 SKU для каждого клиента
- 📊 **ML-моделирование:** LightGBM с калибровкой вероятностей (Platt scaling)
- 🔄 **Модульная архитектура:** Легкое добавление и тестирование новых моделей
- 🧪 **A/B тестирование:** Автоматическое сравнение версий моделей
- 📈 **Backtesting:** Ретроспективное обучение на 26+ месяцах истории
- 🛡️ **Защита данных:** PostgreSQL + защищенные Excel-отчеты

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка базы данных

1. Установите PostgreSQL 14+
2. Создайте базу данных `project_zzz_db`:
```sql
CREATE DATABASE project_zzz_db;
```
3. Примените схему:
```bash
psql -U postgres -d project_zzz_db -f config/schema.sql
```

### 3. Конфигурация

Отредактируйте `config/config.yaml`:
```yaml
database:
  host: localhost
  port: 5432
  name: project_zzz_db
  user: postgres
  password: ВАШ_ПАРОЛЬ  # 🔧 Замените на свой пароль!
```

### 4. Загрузка данных

Поместите исходные файлы в `data/raw/`:
- `Nomenclature.xlsx` — справочник товаров
- `Clients.xlsx` — справочник клиентов
- `Sales_History.xlsx` — история продаж (26+ месяцев)
- `Stock_Remains.xlsx` — остатки на складах
- `Substitutes.xlsx` — замены и аналоги

Запустите загрузку:
```bash
python src/load_references.py
python src/load_sales.py
python src/load_stocks.py
```

### 5. Запуск генерации рекомендаций

```bash
python src/generate_recommendations.py
```

Результат: `data/output/recommendations_YYYY-MM-DD.xlsx`

## 📐 Бизнес-правило 2+2+1

Каждое предложение формируется по правилу:

| Тип | Кол-во | Критерии отбора |
|-----|--------|-----------------|
| **Новые** | 2 SKU | Клиент никогда не покупал, P(buy) > 0.8 |
| **На развитие** | 2 SKU | Из привычных, маркетинговые группы с трендом > +0.05 |
| **На возврат** | 1 SKU | Из привычных, группы с трендом < -0.05 |

**Окно покупки:** [День визита, День визита + 10 дней]

## 🤖 Модели машинного обучения

### Текущая активная модель: `model_lightgbm_v1`

| Метрика | Значение |
|---------|----------|
| Precision@5 | 0.42 |
| Hit Rate | 0.68 |
| Brier Score | 0.15 |
| Время обучения | 2.0 часа |

### Признаки (Features)

**Клиентские:**
- `frequency_30d`, `frequency_90d` — частота покупок
- `avg_check_trend` — динамика среднего чека
- `seasonality_index` — сезонность
- `portfolio_diversity` — разнообразие категорий

**Товарные:**
- `global_popularity` — популярность среди всех клиентов
- `margin_coeff` — маржинальность
- `stock_level` — уровень остатков
- `is_new_flag` — новинка каталога

**Динамические:**
- `days_since_last_purchase_sku` — давность покупки SKU
- `rolling_sales_2w/4w/8w` — скользящие средние продаж
- `cannibalization_score` — корреляция с другими покупками

**Групповые:**
- `group_trend_6m` — тренд маркетинговой группы
- `group_share_in_portfolio` — доля группы в покупках клиента

**Эмбеддинги (ALS):**
- `client_vec_1..50`, `sku_vec_1..50` — векторные представления

## 🧪 A/B тестирование

Система поддерживает автоматическое тестирование новых моделей:

```yaml
ab_test:
  enabled: true
  test_group_ratio: 0.5
  promotion:
    min_duration_days: 14
    significance_level: 0.05
    min_uplift: 0.03  # Мин. улучшение 3%
    auto_promote: true
  degradation:
    critical_threshold: 0.05  # Падение ≥5% = откат
    action: "rollback"
```

### Критерии переключения моделей

**Автоматическое продвижение (staging → production):**
- precision_5 улучшился на ≥ 3%
- hit_rate улучшился на ≥ 3%
- brier_score не ухудшился
- training_time не увеличился более чем в 2 раза

**Критический алерт (требуется откат):**
- precision_5 упал на ≥ 5%
- hit_rate упал на ≥ 5%
- brier_score ухудшился на ≥ 10%

## 📁 Формат выходного файла

Файл: `data/output/recommendations_YYYY-MM-DD.xlsx`

### Лист "Recommendations"

| Поле | Описание | Пример |
|------|----------|--------|
| visit_date | Дата визита | 2024-03-15 |
| client_id | Код клиента | CL-10234 |
| client_name | Название клиента | ООО "Ромашка" |
| manager_id | Менеджер | MGR-02 |
| sku_1 ... sku_5 | Рекомендованные SKU | "ART-5521" |
| type_1 ... type_5 | Тип (new/develop/retain) | "new" |
| group_1 ... group_5 | Маркетинговая группа | "Бренд_Х" |

- **Summary** — статистика генерации
- **Errors** — клиенты без полного набора (если есть)
- **Model Info** — информация о модели
